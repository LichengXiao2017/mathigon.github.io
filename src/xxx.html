Introduction

All functions and methods are part of the M object. Classes are capitalised, but you don't need the new keyword to create instances.

The Fermat library does no parameter and error checking. If you pass in invalid values the behaviour is undefined and functions may throw unpredictable errors.

The Fermat library requires Mathigon’s core.js, which is already included in the minified build above.

Most functions have no or very few dependencies, so it is easy to find and copy particular algorithms from the source code on GithHub.

Core.js extends the native JavaScript Array prototype.

Currently, fermat.js can only use JavaScript numbers, and is restricted to their size and precision. In the future, we will create BigNum and arbitrary precision number classes that work with all these functions.

Arrays

M.tabulate

M.tabulate(Function fn, Integer x1, Integer x2, ..., Integer xn)

creates an n-dimensional matrix that contains fn(i1, i2, ..., in) in cell [i1][i2]...[in].

M.list

M.list(Integer a)

creates an array with values ranging from 0 to a (inclusive)

For example, M.list(5) = [0, 1, 2, 3, 4]
